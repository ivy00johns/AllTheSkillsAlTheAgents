# Installer Test Suite

Tests for `scripts/convert.sh`, `scripts/install.sh`, and `scripts/lint-skills.sh`.

## Prerequisites

Install bats-core:

```bash
# macOS
brew install bats-core

# Ubuntu / Debian
sudo apt install bats

# From source (any platform)
git clone https://github.com/bats-core/bats-core.git /tmp/bats-core
bash /tmp/bats-core/install.sh /usr/local
```

Python 3 with PyYAML is required (used by the scripts under test):

```bash
pip3 install pyyaml
```

## Running Tests

From the repo root:

```bash
# Run the full suite
bash tests/installer/run-tests.sh

# Or invoke bats directly
bats tests/installer/bats/

# Run a single test file
bats tests/installer/bats/01-frontmatter-helpers.bats

# Filter by file prefix
bash tests/installer/run-tests.sh 05
```

## Test Files

| File | What it covers |
|---|---|
| `01-frontmatter-helpers.bats` | `get_field`, `get_body`, `get_array`, `get_owns_dirs`, `fm_check`, `fm_has_field`, `slugify` |
| `02-convert-per-tool.bats` | Per-tool output verification for all 11 tools (paths, frontmatter, stripping warnings) |
| `03-convert-references-handling.bats` | References copy-alongside vs inline-bundle vs skip per tool |
| `04-convert-determinism-and-skip.bats` | Two-run byte-identical output; `requires_claude_code` skip behavior; golden file diffs |
| `05-lint-rules.bats` | All lint rules: errors, warnings, cross-skill checks, JUnit output |
| `06-install-dry-run.bats` | `--dry-run` output, preflight failures, conflict handling, symlink skip |

## Fixture Skills

Located under `tests/installer/fixtures/skills/`. These are minimal synthetic skills that exercise specific code paths:

| Fixture | Purpose |
|---|---|
| `roles/minimal-agent` | Only required fields |
| `roles/full-agent` | Every documented frontmatter field |
| `roles/with-references` | References copy/inline/skip testing |
| `meta/cc-only` | `requires_claude_code: true` skip behavior |
| `meta/no-headers` | openclaw no-headers fallback |
| `meta/long-desc` | 800-char description (WARN-only per v1.1.0 contract) |
| `roles/duplicate-name-a` + `duplicate-name-b` | Cross-skill name collision |

Malformed fixtures for lint error tests live in `tests/installer/fixtures/malformed/`.

## Golden Files

Located under `tests/installer/golden/`. Generated by running `convert.sh` against the fixture skills and committing the output. The `antigravity` golden files have `date_added` normalized to `'YYYY-MM-DD'` because that field is volatile.

To regenerate golden files:

```bash
# From repo root
FAKEREPO=$(mktemp -d)
cp -r scripts "$FAKEREPO/"
cp -r tests/installer/fixtures/skills "$FAKEREPO/skills"
bash "$FAKEREPO/scripts/convert.sh" --out tests/installer/golden 2>/dev/null

# Normalize date_added in antigravity output
find tests/installer/golden/antigravity -name "SKILL.md" | while read f; do
  sed -i "s/^date_added: '[0-9-]*'/date_added: 'YYYY-MM-DD'/" "$f"
done
rm -rf "$FAKEREPO"
```

## CI Notes

The test suite is designed to complete in under 60 seconds on a typical laptop. If it takes longer, `run-tests.sh` will print a warning (but still exit 0 if tests pass).

In CI (non-TTY), `run-tests.sh` outputs TAP format, which GitHub Actions can display as structured test results.
